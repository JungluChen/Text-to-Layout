## How adapters discover tools

All adapters now call `text_to_gds.tool_discovery.tool_paths()` with `None` as the default executable parameter. Discovery order:

1. Environment variable (`TEXT_TO_GDS_JULIA`, `TEXT_TO_GDS_OPENEMS`, etc.)
2. `.tools/<name>-*/bin/<executable>` glob (newest version wins)
3. `shutil.which()` (system PATH)
4. Return `None` → adapter returns `status=SKIPPED`

No hard-coded paths anywhere in adapter code.
