# Quarter-wave CPW resonator — a real Palace run, honestly not verified

A genuine Palace 0.16 execution on an actual cQED device: a λ/4 coplanar-waveguide
resonator on silicon, shorted at one end, open at the other, inside a PEC box.
Three mesh levels ran, the mode was tracked across them by field overlap, and the
result is **`SIMULATION_EXECUTED`, not `PHYSICS_VERIFIED`** — for two independent
reasons, either of which alone would be sufficient.

## What ran

| | |
|---|---|
| Solver | Palace `0.16.0-34-gea2e7b23`, serial, Nédélec order 1 |
| Container digest | `sha256:6d59c1aca1425bcdc71e84502f88a9018689c94fa8bb41476617dbe5ed9509ac` |
| Mesher | Gmsh 4.15.2, unstructured, **non-nested**, local refinement at metal edges |
| Configuration | generated by `textlayout.fem.FEMModel.to_palace_eigenmode_config()` |

Geometry, in millimetres: `L = 5.006564` (λ/4), centre width `0.200`, gap `0.120`,
box `x ∈ [0, 6.20]`, `y ∈ [±1.50]`, silicon `z ∈ [-0.50, 0]` (εᵣ = 11.45), vacuum
`z ∈ [0, 0.50]`. The trace meets the `x = 0` PEC wall — that is the short. The
open end is a `0.120 mm` gap to the end ground plane.

Local refinement is not a luxury here. The field is singular at the conductor
edges, so a uniform characteristic length that resolves the 120 µm gaps would
need an unaffordable element count across the 6.2 mm box. A Gmsh
`Distance` + `Threshold` field takes the mesh from `0.40 mm` in the bulk to
`0.045 mm` at the metal edges.

## Why this is not PHYSICS_VERIFIED

**1. The convergence order is not established.** The declared formal order is 2
(Nédélec p = 1 ⇒ eigenvalue error ~ h²). The observed order is 3.67. The meshes
are unstructured and **non-nested** — Gmsh regenerates from scratch at each
level — and the field is singular at the conductor edges, so the three levels do
not follow one power law. `estimate_order` therefore reports
`in_asymptotic_range = False` and **withholds Richardson extrapolation**. The GCI
of 0.688 % is reported, but it rests on an order that has not been established, so
it is a number rather than a bound. The finest-level change of 0.6975 % also
exceeds the 0.5 % requirement.

**2. The target is a model, not a closed form.** λ/4 with
`ε_eff = (1 + εᵣ)/2 = 6.225` predicts exactly 6 GHz *by construction* — that is
how `L` was chosen. But that formula assumes a thick substrate and an open half
space. This structure has a 0.5 mm substrate and a PEC lid 0.5 mm above the
metal, both of which pull field into the silicon and lower the resonance. Palace
gives `5.709556 GHz`, **−4.84 %** from the model.

Whether that 4.84 % is *model* error or *discretisation* error cannot be
separated until the discretisation error is bounded — and by point 1, it is not.
Calling this verified would attribute the whole discrepancy to the model on no
evidence. The `SIMULATION_EXECUTED` status is not a technicality; it is the
accurate statement.

Contrast the [PEC cavity benchmark](../palace_cavity_te101/README.md), where the
target *is* a closed form, the meshes *are* nested, the observed order matches the
formal one, and the result reaches `PHYSICS_VERIFIED` at `+0.024 %`.

## Acceptance conditions

| Condition | Status |
|---|---|
| a genuine Palace executable or container ran | **met** |
| at least three distinct meshes ran | **met** — 3 |
| element count and DOF increased under refinement | **met** — 64k→192k tets, 94k→267k DOF |
| field-overlap mode match exceeds 0.90 | **met** — min score 0.9595, overlap 1.0000 |
| finest-level frequency change below 0.5 % | **NOT met** — 0.6975 % |
| GCI or extrapolated uncertainty below 1 % | **reported as 0.688 %, but not established** — the order it rests on is not asymptotic |
| domain-size variation below 0.2 % | **not assessed** — a closed PEC box has no truncation boundary. Widening the box would change the answer, and that is untested. |
| energy normalization error below 0.1 % | **met** — participations sum to 1.000000000 |
| participation values converge within 5 % | **inconclusive** — the sequence is monotone and its GCI is 0.018 %, but its observed order (5.23) is no closer to the formal order than the frequency's, and there is no independent value to compare against |
| all inputs, outputs, solver identity and configuration content-hashed | **met** |

## What would fix it

1. **Nested meshes.** Refine by splitting elements rather than regenerating, so
   the three levels describe one power law. This is the single largest cause of
   `p = 3.67`.
2. **A fourth level**, so the order is certified by a refit on the next-coarser
   triple rather than against a declared formal order that the edge singularity
   makes wrong anyway.
3. **A radiation or PML boundary** in place of the PEC lid, which would make
   domain-size convergence meaningful and remove the box loading that accounts
   for most of the −4.84 %.
4. **A better target.** Compare against a converged reference solve or a measured
   device, not against a quasi-static ε_eff whose own accuracy is unquantified.

None of that is done here. The DOF at level C is 267 350; level D at the same
ratio would be ~450 000, and this environment OOM-killed Palace at 795 024.

## Regenerating the evidence

No solver is re-run; the committed outputs are re-parsed with the current parser.

```bash
uv run python scripts/build_palace_benchmark_evidence.py          # write
uv run python scripts/build_palace_benchmark_evidence.py --check  # CI gate
```

Meshes are not committed. Their SHA-256 hashes, element counts and DOF counts are
in `mesh_manifest.json`.
